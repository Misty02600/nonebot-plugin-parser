# [TASK003] - 解析后按需下载媒体资源

**Status:** Pending
**Added:** 2026-03-12
**Updated:** 2026-03-12

## Original Request
当前项目有一个问题，无法让用户自行选择是否要下载视频等资源。添加一个任务，等链接解析完毕后，让用户自行发送“下载视频”或者“下载图片”，来获取连接的资源。

## Thought Process
- 当前插件的核心体验是“发链接即解析”，但媒体资源发送链路偏向自动化，缺少二次确认步骤
- 这个需求不只是新增两个命令，还涉及解析完成后的状态保存和会话级资源定位
- 如果只在渲染后新增命令，而媒体下载任务仍在解析阶段启动，那么用户虽然能“手动领取”，但后台仍然会提前下载资源，不符合按需下载的初衷
- 需要先明确预期行为：
  1. 解析后是否仍然自动发送媒体，还是只发送预览卡片
  2. “下载视频 / 下载图片”应作用于当前用户最近一次解析结果，还是当前群最近一次解析结果
  3. 图文、转发内容、封面图、音频等资源是否纳入按需下载范围

## Implementation Plan
- [ ] 梳理当前 `parser_handler -> renderer.render_messages -> render_contents` 的自动媒体发送链路
- [ ] 明确按需下载的交互边界：预览消息内容、会话缓存粒度、资源类型覆盖范围
- [ ] 设计“下载视频 / 下载图片”对应的 matcher 和解析结果缓存方案
- [ ] 评估是否需要把媒体下载改为真正的延迟触发，而不是仅延迟发送
- [ ] 补充测试并验证群聊/私聊场景
