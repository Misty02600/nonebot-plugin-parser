# Active Context

## 当前工作重点
- 日常使用中发现的 Bug 修复和记录
- 解析器兼容性问题排查
- 媒体资源按需下载交互需求记录
- 直播链接解析扩展需求记录

## 最近发现的问题
1. **B 站专栏链接解析失败** - `bilibili.com/read/cvXXX` 触发 `KeyError: 'readInfo'`，bilibili_api 的 `article.turn_to_opus()` 在某些文章返回数据缺少 `readInfo` 字段
2. **抖音精选链接无法解析** - `douyin.com/jingxuan?modal_id=XXX` 格式未被正则匹配，现有模式仅匹配 `jingxuan.douyin.com/m/video/XXX`
3. **无法让用户按需获取媒体资源** - 当前链接解析后会直接进入媒体资源下载/发送流程，用户无法在解析完成后再手动发送“下载视频”或“下载图片”来获取资源
4. **直播链接覆盖范围和建模都偏弱** - 当前仅 B 站有直播解析入口，且直播结果仍走通用卡片模型；其它已支持平台暂无直播专用 handler

## 近期决策
- 暂不修改代码，先记录为任务跟踪
- 新的按需下载诉求先登记为任务，后续再分析交互流程和实现边界
- 直播解析先以“房间预览元数据”为第一阶段目标，B 站优先，其它平台按可行性逐步接入
- 直播不再继续视为普通动态/视频结果，后续应新增独立分类并配套专用渲染方法
